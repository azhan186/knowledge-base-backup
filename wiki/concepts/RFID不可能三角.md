# RFID 不可能三角

**Impossible Triangle of RFID Design**：读取距离 / 功能复杂度 / 成本 **三者不可同时兼得**——这是 RFID 芯片设计的根本约束。

## 三角的三个顶点

| 顶点 | 含义 |
|------|------|
| **读取距离** | 标签能多远被读到（UHF 几米 / HF 厘米）|
| **功能复杂度** | 标签能做什么（只读 ID / 加密 / 传感器）|
| **成本** | 单价（几分钱 / 几毛钱 / 几美元）|

## 物理根源

```
P_chip ∝ 逻辑功能复杂度
P_recv ∝ 1/d⁴
```

- **功能多 → 芯片功率高 → 距离短**
- **距离远 → 能量预算紧 → 功能必须少**
- **成本低 → 芯片面积小（晶体管少）→ 功能受限**

## 现实案例

| 场景 | 距离 | 功能 | 成本 |
|------|------|------|------|
| 物流标签 | 几米 | 只读 ID | 几分钱 |
| 服装零售 | 几米 | EPC | 几分钱 |
| 门禁卡 | 厘米 | 简单加密 | 几毛钱 |
| 公交卡 | 厘米 | 安全认证 | 1 元+ |
| 支付卡 | 厘米 | 强加密+存储 | 5 元+ |
| 电子护照 | 厘米 | 多层认证+生物 | 数十元 |

## 设计师的"妥协艺术"

- 选 UHF + 几分钱 = **牺牲功能换距离和成本**（物流）
- 选 HF + 几毛钱 = **牺牲距离换成本和功能**（门禁）
- 选 UHF + 强加密 = **牺牲成本换距离和功能**（特殊场景）
- **三全其美：不存在**

## 突破尝试

| 方向 | 思路 | 现状 |
|------|------|------|
| 能量采集 | 用更强阅读器供能 | 距离受法规限制 |
| 计算式标签 | 把功能转移到阅读器 | 前沿研究 |
| 柔性电池辅助 | 牺牲部分"无源"换功能 | 商业化中 |
| 多阅读器协同 | 阅读器端增加算力 | 实施复杂 |

## 关键洞察

> **不可能三角 = 物理 + 经济 + 工程的综合约束**
> 不试图打破它，**而是在三角内优雅权衡**——这是 RFID 工程师的核心能力。
