# AI-Agent（人工智能智能体）

> 核心定义：AI Agent是能够自主感知环境、做出决策并采取行动以达成目标的程序或系统。

## 三个核心能力

| 能力 | 说明 |
|------|------|
| **感知** | 通过传感器、数据输入等方式获取信息 |
| **决策** | 基于目标和当前信息，规划行动 |
| **行动** | 执行具体操作 |

## 传统软件 vs AI Agent

| 特征 | 传统软件 | AI Agent |
|------|----------|-----------|
| 自主性 | 需要人每一步下指令 | 不需要人每一步下指令 |
| 反应性 | 静态 | 能根据环境变化调整 |
| 目标导向 | 固定流程 | 为完成明确目标而工作 |

> **大语言模型 vs AI Agent**：纯大模型只会"对话"，AI Agent像给大模型装上了手和脚——可以自己查资料、调用工具、执行任务、反复试错。

## 生活类比：能干的私人助理

场景：订下周去北京的高铁票，预算500元以内，尽量靠窗。

1. **感知**：打开12306或携程，查看车次和余票
2. **决策**：比较价格、时间、靠窗座位
3. **行动**：点击购买、支付
4. **反馈**：没票时主动建议改期或加价升等

## 历史演变

| 阶段 | 时间 | 特点 | 例子 |
|------|------|------|------|
| 规则式Agent | 1950s–1980s | 人写死"如果…就…"规则 | 恒温器：温度>30℃就开空调 |
| 反应式Agent | 1980s–1990s | 感知环境并即时反应，无记忆 | Roomba扫地机器人：撞墙就转弯 |
| 基于模型的Agent | 1990s–2000s | 内部维护"世界模型"，能推理 | 自动驾驶汽车 |
| 目标驱动/效用驱动 | 2000s–2010s | 比较不同行动路径，选最优 | AlphaGo |
| 学习型Agent（现代） | 2010s–今 | 结合强化学习、大语言模型 | ChatGPT + 插件 |

## 技术层面的三种方式

| 方式 | 做法 | 影响 |
|------|------|------|
| **外部记忆检索**（最常用） | 对话记录存进向量数据库，每次提问检索最相关片段作为"提示背景" | Agent没有真正改变模型，只是多拿一些上下文。优点是灵活安全，缺点是无法深入归纳总结 |
| **微调（Fine-tuning）** | 用历史对话数据调整模型的的部分参数 | Agent真正永久改变行为，更适应你的语气和知识。但成本高，存在"灾难性遗忘"风险 |
| **持续学习 / 元学习** | Agent自己写"总结笔记"存入记忆，依据笔记调整推理策略 | 类似"反思日记"，可相对轻量改进，但仍有反馈偏差风险 |

## 私人知识库对AI-Agent的影响

| 积极影响 | 说明 |
|----------|------|
| 高度个性化 | 记住偏好，自动匹配风格 |
| 避免重复提问 | 长线任务能记住人物名称、章节大纲，不会前后矛盾 |
| 错误自我修正 | 纠正过的错误会存入知识库，下次直接给正确版本 |
| 学习新知识 | 可成为特定领域"专家" |

| 负面影响与风险 | 说明 |
|----------------|------|
| 过拟合到你的噪音 | 矛盾信息会导致Agent混乱 |
| 隐私泄露 | 知识库含生活习惯、健康、财务信息，被攻击则全部外泄 |
| 形成偏见/回声室 | 只学习你一个人的数据，丧失客观性 |
| 遗忘必要的一般知识 | "再学习"权重过高可能覆盖正确基础常识 |
| 行为不可预测 | 积累多了无法精确知道Agent学到了什么"隐含规则" |

## 与《维度黎明》的关联

- **AI Agent的自主性** → 可用于设计《维度黎明》中融合者或智能体的自主行为
- **外部记忆检索 vs 微调** → 可用于设计维度中的记忆存储机制
- **"灾难性遗忘"** → 可以作为维度中记忆冲突的隐喻
