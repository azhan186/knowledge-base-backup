# 挑战-应答认证协议

**Challenge-Response Authentication Protocol**：通过让标签用**密钥 + 随机数**计算响应，**不直接暴露密钥**的认证机制。**这是防止 RFID 标签克隆的标准方法**。

## 协议流程

```
阅读器 → 标签：随机数 R（挑战）
标签：响应 = f(K, R)（K 是标签密钥）
标签 → 阅读器：响应
阅读器：用相同 K 和 R 独立计算 f(K, R)
阅读器：比对响应是否一致
```

## 关键特性

- **不直接传输密钥**：即使攻击者截获响应，也无法反推 K（前提是 f 是单向函数）
- **每次不同**：R 是随机的 → 每次响应都不同 → **无法重放攻击**
- **双向认证**：可让阅读器也证明自己（避免被恶意阅读器读取）

## 实际部署

| 场景 | 协议 | 密钥长度 |
|------|------|----------|
| 物流标签 | 无 | 0（明文 ID）|
| 门禁卡 | Mifare Crypto-1（已破）| 48 位 |
| 电子护照 | BAC/PACE | 128 位 |
| 支付卡 | 3DES/AES | 128 位+ |
| 高端证卡 | RSA/EC | 2048 位+ |

## 矛盾：成本 vs 安全

- **一次性物流标签**：单价几分钱，承担不起强密码电路
- **支付卡/护照**：单价数美元，可内嵌复杂密码协处理器
- **这就是为什么不可能三角存在**——[[RFID不可能三角]]

## 进阶：主动认证 vs 被动认证

- **被动认证**：阅读器发起，标签响应（标准挑战-应答）
- **主动认证**：标签也能反向挑战阅读器（防"恶意阅读器"读取）
- 详细对比见 [[主动被动认证]]
