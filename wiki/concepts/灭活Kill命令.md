# 灭活（Kill）命令

**Kill Command**：EPC Gen2 协议中携带 **32 位密码**的不可逆自毁命令。**这是 RFID 隐私保护的终极手段**，也是 Auto-ID 中心早期为应对隐私恐慌而设计的伦理承诺。

## 工作机制

1. 零售结账后，POS 发出携带 32 位密码的 Kill 指令
2. 标签芯片验证密码
3. **永久自毁**——后续任何阅读器都无法再读取
4. 标签物理上还在，但**功能上等于报废**

## 关键限制

### 1. 不可逆
- 一旦激活，**无法再利用**（退货、家庭应用）
- 许多场景改用"休眠"或"访问密码锁定"来临时屏蔽

### 2. 32 位密码太弱
- 暴力破解 = 2^32 ≈ 43 亿次尝试
- 在专业设备下可能几小时到几天内破解
- **更高安全性的标准已制定**（128 位加密等），但快消供应链普及率不高

### 3. 消费者无法确认
- 普通消费者不知道标签**是否真正被灭活**
- 可能是"永久灭活"或"仅仅设为不可见"
- **这反映出安全需求和成本、便捷之间的持续博弈**

## 伦理转向（专家视角）

- Auto-ID 中心最初设想 = 标签存 EPC 码（指向网络数据库）
- 设计 Kill = **用一个不可逆承诺换公众对技术部署的许可**
- 商业实践发现：退换货追踪、二手市场价值 → 业界偏向**可逆方案**
- **隐私政策模糊 = 早期协商不彻底的产物**
- **未来不确定性**：GDPR 等更严格法规是否强制恢复 Kill 机制，尚未可知

## 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Kill 命令** | 绝对保护 | 不可逆 |
| **休眠** | 可逆 | 安全等级低 |
| **访问密码** | 可逆、灵活 | 需管理密码 |
| **物理屏蔽**（法拉第笼）| 无电子 | 损耗便利性 |
| **签名加密**（高级）| 真正安全 | 成本高 |
