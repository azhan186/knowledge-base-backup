# Gen2 四会话层

**EPC Gen2 Four Session Flags**：协议设计的核心抽象之一，**在无状态消息传递之上构建有状态业务流程**的优雅机制。

## 四个会话层

| 会话 | 盘存标志持久性 | 典型用途 |
|------|----------------|----------|
| **S0** | 短暂（掉电消失）| 快速临时盘存 |
| **S1** | 短暂 | 单次操作 |
| **S2** | 持久（掉电保持）| 跨操作追踪 |
| **S3** | 持久 | 长期状态记录 |

## 经典工作流

```
阅读器 A：Query → 标签进入 S0 标志（短暂）
  ↓
阅读器 A：把所有 S0 标签搬到 S2（持久）
  ↓
阅读器 B：Query 只关心 S2 标签
  ↓
细读 → 读完搬回 S0（重置）
```

## 核心抽象

- **持久性梯度**：不同会话有不同寿命，满足不同业务流程
- **无状态消息 + 有状态业务**：阅读器不需要记录每个标签的完整状态，只通过会话标志位协调
- **多阅读器协作**：让分工流水线化（粗盘点 → 细读 → 状态转移）

## 专家视角

> **Gen2 = 对分布式系统"最少承诺"哲学的完美实践**

复杂度留在阅读器，简单留给标签。4 会话层是**极少数**允许标签承担少量状态的设计，但通过"持久性梯度"做了严格控制。

## 实际应用

- 仓储多阅读器协作（入口盘点 → 库内细读 → 出口核验）
- 零售多阶段处理（收货 → 上架 → 销售）
- 酒店布草盘点（粗清点 → 洗涤流转 → 库存核验）
