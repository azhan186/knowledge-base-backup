# LDPC 纠错码

**Low-Density Parity-Check（低密度奇偶校验）码**：一种逼近香农极限的信道编码，是现代 HDD / SSD / 5G / Wi-Fi 的核心纠错技术。

## 核心地位

- 1962 年由 Robert Gallager 提出
- 沉寂 30 年后因 Turbo 码重燃信道编码研究兴趣
- 1990 年代被"重新发现"，现已成为**HDD 读通道**、5G NR、Wi-Fi 6/7、DVB-S2 等的事实标准
- 与 PRML/SOVA 配合，让 HDD 能在极低 SNR 下可靠还原数据

## 基本思想

### 1. 码字结构：稀疏校验矩阵

- LDPC 码由一个**巨大的稀疏矩阵 H** 定义（"低密度"指矩阵中 1 的占比极低）
- 合法码字 c 满足 H·cᵀ = 0
- 矩阵的稀疏性 → 解码可高效并行

### 2. 软判决 vs 硬判决

- 硬判决：解调器直接给 0/1
- **软判决**：解调器给**每比特是 1 的概率**（对数似然比 LLR）
- LDPC 解码器用**软信息**迭代修正，比硬判决性能高数 dB

## 迭代解码（Belief Propagation / 和积算法）

### 单轮迭代步骤

1. **变量节点 → 校验节点**：每个变量节点（比特）把自己当前对 1/0 的"信念"告诉所有相连的校验节点
2. **校验节点 → 变量节点**：每个校验节点根据"哪些邻居都说 1/0 时这个等式能成立"，计算**邻居应该修正成什么**
3. **更新信念**：变量节点综合所有校验节点的反馈，更新自己的 1/0 信念
4. **重复**，直到所有校验方程都满足（或达到最大迭代次数）

### 与 SOVA 通道检测器的联合迭代

HDD 读通道的工程实现：

```
SOVA 通道检测器 → 给每比特软信息（置信度）
        ↓
LDPC 解码器 → 找不满足校验的可疑比特 → 修正后的概率
        ↓
反馈给 SOVA → 通道检测器重新评估波形序列
        ↓
      循环数轮
        ↓
所有奇偶校验通过 → 比特序列确认
```

**形象比喻**：两个专家（通道检测器 + 纠错码）轮流修正对方的判断，直到共识。

## 性能特点

- **逼近香农极限**：距离理论极限仅 0.0045 dB（已知最佳码之一）
- **长码字性能极佳**：块长越长越接近极限
- **并行友好**：稀疏矩阵 + 局部消息传递，可高度并行硬件化
- **延迟可控**：迭代次数有界（HDD 典型 5-15 轮）

## 应用领域

- **HDD 读通道**（与 SOVA 迭代）
- **SSD 控制器**（NAND 闪存纠错）
- **5G NR 数据信道**
- **Wi-Fi 6/7**
- **DVB-S2 卫星电视**
- **以太网 802.3**
- **深空通信**（CCSDS 标准）

## 关键人物

- **Robert Gallager（1962）**：博士论文首次提出
- **David MacKay、Radford Neal（1990s）**：复兴并推广
- 2015 年 Erdal Arıkan 的 Polar 码是 5G 控制信道标准，但 LDPC 仍是数据信道主流
