# 技术路径依赖

**Technological Path Dependence**：技术选择**并非全然理性**——早期决策、关键事件、偶然性都会**锁定**后续发展路径，使替代方案越来越难以追赶。

## RFID 中的两大典型案例

### 案例一：UHF vs 2.45 GHz 之争

**1990 年代末**：
- 2.45 GHz 候选：更小天线 + 更大带宽
- UHF 候选：更远距离 + 更强穿透

**关键物理原因**（UHF 胜出）：
- 2.45 GHz 对水/材料**穿透损耗远高于 UHF**
- 短波长 → **多径衰落更剧烈**
- 密集货箱中**相干距离极短** → 容易形成驻波空洞

**结论**：**不是某一个指标的胜出，而是对整个复杂工作场景适配性的综合胜出**。

### 案例二：Kill 命令的伦理转向

**Auto-ID 中心最初设想**：
- 标签存 EPC 码（指向网络数据库）
- 设计 Kill 命令 = **不可逆自毁承诺**
- 目的：换公众对技术部署的许可

**商业实践发现**：
- 永久退换货追踪的需求
- 二手市场价值
- 业界偏向**可逆方案**（休眠、访问密码）

**现在的模糊地带**：
- 消费者不知道标签是"真灭活"还是"仅不可见"
- **早期协商不彻底的产物**
- **未来不确定性**：GDPR 等更严格法规是否强制恢复 Kill 机制，尚未可知

## 路径依赖的机制

```
关键决策点 A
  ↓
技术生态围绕 A 优化（标准、芯片、应用、用户习惯）
  ↓
替代方案 B 越来越难追赶
  ↓
路径锁定
```

## 摆脱路径依赖的难点

- **沉没成本**：已部署的数十亿 EPC Gen2 标签无法一夜更换
- **网络效应**：标准、读卡器、应用软件都已固化
- **既得利益**：现有玩家没有动力改变
- **认知惯性**：用户/工程师习惯现有方案

## 关键洞察

> **技术选择 = 理性 + 偶然性 + 沉没成本 + 网络效应的合成**
> 看懂技术史，需要**回溯关键决策点**——而不是只看"赢家通吃"的现状。
