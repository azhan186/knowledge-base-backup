# Q 算法（动态帧时隙）

**Q Algorithm (Dynamic Framed Slotted Aloha)**：EPC Gen2 协议中用于**多标签盘存**的反馈控制算法。**不预测标签数量，而是像控制恒温器一样"调整"它**。

## 算法流程

1. **阅读器发 Query**（含 Q 值），标签从 0 到 (2^Q - 1) 随机选一个时隙
2. **观察本轮结果**：
   - 大量**碰撞**（多人同时回答）→ 标签稠密，帧太小 → **Q+1**（时隙翻倍）
   - 大量**空闲**（无人应答）→ 标签稀疏，帧太大 → **Q-1**（时隙缩小）
   - 适度应答 → 保持 Q
3. **下一轮 Query 重复此过程**

## 理论极限

- 纯时隙 Aloha 最大信道利用率：**G·e^(-G) = 36.8%**（G=1 时）
- 也就是说即使在最理想情况下，**63.2% 的时隙被浪费**（碰撞或空闲）
- 实际系统可达 30-35% 已经很优秀

## 控制论视角

- **状态变量**：标签数量（不可直接观测）
- **观测变量**：本轮碰撞/空闲/应答次数
- **控制动作**：Q+1 / Q-1 / 保持
- **目标**：逼近 36.8% 利用率

> 这是一个**经典的反馈控制过程**，以极低的计算开销，动态逼近系统吞吐量的理论最大值。

## 工程师的优化空间

- **Q 值步进大小**：1 / 2 / 4
- **平滑滤波**：本轮 + 历史 N 轮加权
- **提前终止**：连续多轮无新标签则停
- 这些微调能在**几毫秒**的盘存周期内**榨取最后几个百分点的效率**

## 实际应用

| 场景 | Q 范围 | 时隙数 |
|------|--------|--------|
| 零售单件 | 0-2 | 1-4 |
| 货架盘点 | 2-5 | 4-32 |
| 仓库托盘 | 5-8 | 32-256 |
| 物流货箱 | 8-12 | 256-4096 |
| 工业密集 | 12-15 | 4096-32768 |
