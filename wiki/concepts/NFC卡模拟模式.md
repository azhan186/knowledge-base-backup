# NFC 卡模拟模式

**NFC Card Emulation Mode**：让**手机**在外部阅读器看来**像一张普通 HF RFID 标签**的工作模式。**这是 NFC 技术的"皇冠"应用**——手机在物理上变成一张无源 HF 标签。

## 三种 NFC 模式

| 模式 | 角色 | 巧思 |
|------|------|------|
| **读写器** | 主动 | 与普通 RFID 阅读器无异 |
| **卡模拟** | 被动 | 手机物理上变无源 HF 标签 |
| **点对点** | 对等 | 两个设备都能主动载波 |

## 卡模拟的精妙机制

### 硬件组成
- **NFC 控制器**：管理射频信号
- **安全元件（SE）**：存密钥和敏感数据（独立芯片）
- **天线**：与外部阅读器耦合

### 工作流程

```
1. 手机靠近外部阅读器
2. NFC 控制器检测到外部载波场
3. NFC 控制器将"能量+信号"路径完全交给 SE
4. 手机系统 CPU 可以是关机/休眠状态
5. SE 通过天线与外部阅读器通信
6. 外部阅读器认为它在与一张普通 HF 标签通信
```

## 关键洞察

> **手机在物理上就变成了一张纯粹的无源 HF 标签**
> 即使手机没电、关机，仍可刷卡

## 实际应用

- **Apple Pay / Samsung Pay / 华为支付**：把银行卡信息存在 SE 中
- **公交卡 / 门禁卡**：把卡片信息迁移到手机 SE
- **身份证电子化**：部分国家支持
- **会员卡 / 积分卡**：替代实体卡

## 与传统 RFID 的边界突破

| 传统 RFID | NFC 卡模拟 |
|----------|-----------|
| 严格的"一问一答" | 手机可以主动决定何时响应 |
| 标签是被动设备 | 手机是被"伪装的"标签 |
| 标签无操作系统 | SE 内部可有完整 JavaCard 操作系统 |
| 物理形态固定 | 可显示动态二维码/状态 |

## 安全机制

- **SE 物理隔离**：即使手机被黑，SE 不受影响
- **TEE（可信执行环境）**：主系统与 SE 之间的安全桥梁
- **Tokenization**：每次交易用一次性令牌，真卡号不暴露
- **生物认证**：支付时验证指纹/面容
