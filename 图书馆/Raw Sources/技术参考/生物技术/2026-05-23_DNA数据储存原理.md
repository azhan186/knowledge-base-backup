## 原理：数字信号 → DNA字母

**编码对应规则：**
- 00 = A
- 01 = C
- 10 = G
- 11 = T

**写入流程：**
1. 数字数据（0和1）按规则转化为DNA字母序列（A/T/C/G）
2. 通过DNA合成仪在实验室"打印"出这条序列，得到DNA粉末
3. 储存

**读取流程：**
1. 用基因测序仪读出A、C、G、T的排列顺序
2. 按规则转回0和1，还原数据

## 为什么诱人？

| 优势 | 数据 |
|------|------|
| **存储密度** | 1克DNA ≈ 2.15亿GB，全世界数据可装进约一纸杯 |
| **寿命** | 阴凉干燥处可保存成千上万年（猛犸象化石DNA仍可读取） |
| **格式永不过时** | 只要人类还在研究生物学，就永远有测序仪读取它 |
| **无需供电** | 不依赖能源，不像硬盘或磁带 |

对比：硬盘几年可能损坏，磁带十几年老化。

## 当前瓶颈

| 瓶颈 | 说明 |
|------|------|
| **成本极高** | 读写一部高清电影的成本是天文数字 |
| **速度极慢** | 合成和测序耗时且昂贵 |
| **无法随机存取** | 读文件某一页可能需要测序整管DNA，像顺序读取的磁带 |
| **错误率** | 合成和测序过程会产生字母错误，需要纠错算法 |

---

## 现状与前景

- **目前定位**：长周期、冷数据的"归档存储"（国家档案、文化遗产等）
- **已验证**：微软等公司已成功在DNA中储存《OK Computer》专辑等数据
- **短期应用**：先瞄准归档存储，短期内无法替代硬盘
