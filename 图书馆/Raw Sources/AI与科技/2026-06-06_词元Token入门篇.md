# 词元（Token）入门篇

## 核心定义

在大语言模型和自然语言处理中，词元（token）是文本被切分之后的最小处理单元。

可以把模型想象成一个只认识数字的加工机，我们无法直接把汉字或英文喂给它。所以一段文字需要先被"切"成一个个小片段，每个小片段就是一个 token。然后每个 token 会被分配一个独一无二的数字编号，模型真正运算的就是这些编号序列。

一个 token 的大小并不固定：
- 一个完整的常用词，如 "the"
- 一个词的组成部分（子词），如 "play" 和 "ing"
- 一个单独的汉字、标点，甚至一个空格

【可靠度评价】这是当前大语言模型领域的基石概念，定义清晰且共识度极高，可靠度很高。

## 生活类比：厨师小明的肉丁

假设有一个叫小明的厨师，他只擅长烹饪大小固定的"一口肉丁"。你给了他一大块五花肉和一张菜谱（你的输入文本）。

小明没法直接烹饪整块肉，必须先把肉切成大小均匀的小肉丁。切好后，他给每块肉丁贴上标签编号（如"瘦肉丁-001"、"肥肉丁-002"、"带皮肉丁-003"）。然后，他完全不看肉的原样，只按照标签编号的顺序，一口一口地把肉丁做成菜。

- 整块五花肉 = 你输入的一段话
- 每块切好的小肉丁 = 一个词元（token）
- 标签编号 = 词元在模型词表中的唯一 ID
- 小明这个厨师 = 大语言模型

模型根本不"读"字，它读的是一串代表"肉丁"的编号，并预测下一块最可能是什么编号的肉丁。

【可靠度评价】这是一个基于成熟知识的虚构类比，用于直观理解，可靠度很高。

## 历史演变

词元概念的演变，就是人们不断寻找"最合适的文本切分大小"的过程。

### 词级分词（早期）
最自然的想法是用空格和标点把文本切成词。如"我喜欢猫"切成"我/喜欢/猫"。这很简单，但会遇上词典爆炸问题（全世界语言词汇量太大）和集外词问题：模型一旦遇到没见过的词就无法处理。

### 字符级分词
一刀切到最小，每个字母或汉字都是一个 token。如 "cat" 变成 "c" "a" "t"。这彻底解决了集外词问题，但序列变得极长，模型很难记住远距离的依赖关系，训练也变慢。

### 子词级分词（革命性突破）
这是目前所有主流大模型的基础。核心思想是："把常见词整体保留，把罕见词拆成有意义的零件"。

典型方法如字节对编码（BPE），它从字符开始，反复把最常见的相邻符号合并成一个新符号。比如发现 "t" 和 "h" 经常一起出现，就合并出 "th"；"th" 和 "e" 又常一起出现，就合并出 "the"。这样 "theater" 可能被拆成 "the" 和 "ater"。

这个思路在 2015-2016 年左右被引入机器翻译，并迅速推广到整个领域。

【可靠度评价】这是自然语言处理领域公认的发展脉络，BPE 等方法为成熟技术，可靠度很高。

## 现实应用

词元的概念已经渗透到许多产品和计费逻辑中：

### 大模型的计费与限制单位
几乎所有商用大模型的收费和上下文窗口都是按 token 算的：
- 输入的问题消耗输入 token
- 模型生成的回答消耗输出 token
- 总消耗 token 数 = 输入 token 数 + 输出 token 数

### 内容生成的核心流程
模型在生成回答时，就是一次又一次地重复"根据上文 token 序列，预测下一个最可能的 token 是什么"这个动作。预测出一个 token，就把它附加到序列尾部，再预测下一个，如此循环直到生成出结束符号。

### 文本搜索与处理
在搭建知识库问答时，需要先把大量文档都切成 token，才能建立索引和检索。切分的方式直接影响搜索的准确度。

【可靠度评价】这是大模型产品和应用中的实际运行机制，是成熟的基础知识，可靠度很高。

## 主要争议与前沿挑战

尽管子词分词已非常成功，但它仍然不是完美的解决方案：

### 多语言表现不均
同一个意思，用英文表达可能就几个 token，用中文、日语或黏着语、资源贫乏的小语种表达，可能会被切成多得多的 token。这导致非英语用户在同样长度下，消耗更多费用、能放进上下文窗口的有效信息也更少。这是一种语言公平性问题。

### 对拼写错误和噪声敏感
一个故意的错别字或拼写错误，可能会让分词结果面目全非。如 "loooove" 可能被拆成 "lo" "oo" "ove" 这样毫无意义的碎片。这也是一些对抗攻击的突破口。

### 数值和符号处理困难
数字 "2025" 可能被切成 "20" 和 "25"，让模型难以真正理解其数值大小。这使得大模型在精确数学推理上存在天生短板。

【不确定性】目前业界正在尝试多种新方案（如将数字单独编码、采用字符级或字节级分词回溯），但没有哪种方案被公认为彻底解决了问题，仍处于探索阶段。

### 分词器固化问题
大模型一旦训练好，它的分词器就被固化了。后续哪怕发现了更好的分词方式，也很难在不重新预训练模型的情况下更换它，因为模型权重是和词表 ID 深度绑定的。

### 多模态的统一
在图像、视频、音频等领域，也借鉴了"词元"概念，将图像切成小块（patch）变成"视觉词元"。但如何高效、公平地将不同模态的信息统一成一种通用的"词元"，是重要的前沿挑战。

【可靠度评价】争议点和挑战均来自当前领域内的普遍讨论和实际观察，语言公平性、数值处理等问题是公认的不足，但"最佳解决方案"目前并无定论，这里存在不确定性。
