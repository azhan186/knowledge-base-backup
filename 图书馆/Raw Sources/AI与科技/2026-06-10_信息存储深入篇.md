## 一、深入比特：1 和 0 怎样变成物理世界里的东西

计算机里所有信息——文字、图片、声音、程序——最终都变成 **0 和 1** 组成的串。但 0/1 不是纸上的数字，而是用**真实的物理量**代表：

| 介质 | 物理载体 | "1" | "0" |
|------|---------|-----|-----|
| **闪存** | 微小"杯子" | 装了一定数量电子 | 基本空着 |
| **磁带/硬盘** | 微小磁针 | 指向北极 | 指向南极 |
| **光盘** | 表面小坑/平地 | 弱反射（坑） | 强反射（平地） |

> 🔑 **关键点**：只要能保持状态不动 → 存住；只要能改变状态 → 写入新信息；只要能检测出当前状态 → 读出来。

### 基本单位

- **1 比特（bit）**：一个 0 或一个 1
- **1 字节（Byte）= 8 比特**：可表示一个英文字母
- 例："A" 在计算机里就是 `01000001`（8 比特序列）

> [!info] 标注
> 这是数字电路和计算机系统的基础，可靠度极高。

## 二、深入机械硬盘 vs 固态硬盘：为什么一个怕震，一个怕久放

### 1. 机械硬盘（HDD）

**超高精度留声机**：

- 内部一叠涂满磁层的铝制圆盘，**每分钟 5400 / 7200 转**高速旋转
- 每个盘面上方一根**悬空的磁头**，离盘面近到像波音 747 贴地飞行而不触地
- 写入：磁头产生磁场，改变盘面上某个微区的磁化方向
- 读取：磁头感应磁场变化，转成电信号

| 优点 | 致命弱点 |
|------|---------|
| 成本极低 | **怕震动**——碰撞让磁头撞高速盘面 → 刮掉磁层 → **数据瞬间丢失** |
| 海量存不太动的文件 | — |

### 2. 固态硬盘（SSD）

**高级版记忆水晶，无活动部件**：

- 核心是 **NAND 闪存**，由一个个"**浮栅晶体管**"构成
- 每个晶体管有一个**被绝缘层包裹的小"储物室"**
- 写入：高电压让电子穿过绝缘层**隧穿**进去，困在里面
- 读取：检测电流通断判断状态，完全不碰储物室

| 优点 | 致命弱点 |
|------|---------|
| 极快、抗摔、省电 | **擦写次数有限**（家用 SSD 通常撑几百 TB） |
| — | **长时间不通电 → 电子慢慢漏掉**（高温环境可能 1-2 年就挥发） |

> [!info] 标注
> 工作原理是成熟的工业常识，可靠度很高；数据保持期具体数值受环境影响大。

## 三、深入"为什么买到的容量总比标称小？"

### 两套计数法在打架

| 角色 | 单位 | 进制 |
|------|------|------|
| **制造商** | KB / MB / GB / TB | **十进制**（1 KB = 1000 字节） |
| **操作系统** | KiB / MiB / GiB / TiB | **二进制**（1 KiB = 1024 字节） |

### 算账示例：标称 1 TB 硬盘

```
制造商：1 TB = 1000 × 1000 × 1000 × 1000 字节
     = 1,000,000,000,000 字节

操作系统：1,000,000,000,000 / 1024 / 1024 / 1024
     ≈ 931 GiB（吉基字节）

系统显示"931 GB"（但实际是 GiB）
```

> 字节总数**没变**，只是单位换算。

> [!info] 标注
> 计算机行业固有差异，已存在数十年，**可靠度极高**。

## 四、深入云存储：你的文件到底散落在何方

> "云端"**不是天上的优盘**，而是庞大的**分布式存储系统**。

### 流程（以猫照片为例）

| 步骤 | 动作 |
|------|------|
| **1. 切块** | 把照片切成小块，如每块 4 MB |
| **2. 编码冗余** | 计算出**校验块**（**纠删码**），即使几个块丢失也能用剩余块算出原数据 |
| **3. 四处存放** | 数据块 + 校验块分散到**不同机柜、不同楼层、不同城市**的硬盘上 |
| **4. 元数据管理** | 高速数据库记住"第 37 号块放在深圳 A 机柜第 12 块盘"——访问时瞬间从各地拉取拼回 |

### 本质

> **用大量廉价不可靠的硬件，通过软件堆出一套极其可靠的虚拟大硬盘**。

> [!info] 标注
> 对主流分布式存储架构的简化描述，可靠度高。各厂商具体参数与算法是商业实现细节，存在不确定性。

## 五、深入"数字黑暗时代"：存的不仅是介质，更是整个生态

存几十年难在**三个层面**：

### 1. 介质物理死亡

- 磁盘生霉
- 光盘氧化
- 闪存漏电

### 2. 阅读设备灭绝

- 你现在还能找出**读取 ZIP 磁盘**或**小型 DV 录像带**的设备吗？
- 即便有，接口也早变了

### 3. 文件格式与软件绝迹

> **逻辑失读**：假设 1990 年用某个早期排版软件存的文档，格式专有。今天就算文件完整拷出来，没有那个软件，看到的也是一堆乱码。

> 这就像**古代石刻虽然完好，但全世界已经没人认识那种文字了**。

### 应对思想

> 🔑 **不断迁移 + 多重备份**：每隔几年，把数据复制到当前主流新介质上，并转换成**当时的标准开放格式**。

> 归档不是"**存进去就永远不管**"，而是像**维护花园**一样需要**持续照料**。

> [!info] 标注
> 数字保存是档案学与计算机科学交叉的成熟领域，描述可靠度高，但"多少年必须迁移"无绝对标准。

## 六、深入前沿：DNA 存储怎么用 A/T/C/G 编出电影？

> 用四种碱基 **A、T、C、G** 编码任意数字信息——这是最迷人的方向。

### 原理概要

| 步骤 | 操作 |
|------|------|
| **1. 比特 → 碱基** | 规定 00 → A，01 → C，10 → G，11 → T |
| **2. 合成 DNA** | 化学合成极微小的 DNA 分子 = "文件副本" |
| **3. 长期保存** | DNA 粉末密封在惰性气体中，**阴冷干燥**——理论上完整保存**数百年甚至上千年** |
| **4. 读取** | 测序仪读出碱基序列（A C G T...）→ 按规则反译回 0 和 1 |

### 致命问题

- 现阶段合成 + 测序**成本是天价**
- 读取一瓶数据可能**数小时甚至数天**
- **无法直接定位"第 37 句话"**——只能整瓶读出来再找

### 当前应用

仅限于把**极其重要且极少访问的文化遗产**作为实验存入。

> [!warning] 标注
> 正在活跃研究的前沿领域，原理公认，**实用时间表高度不确定**。

## 七、关键术语（本篇新增）

| 术语 | 含义 |
|------|------|
| **比特（bit）** | 0 或 1 的最小信息单位 |
| **字节（Byte）** | 8 比特，可表示一个英文字母 |
| **浮栅晶体管** | Floating Gate Transistor，闪存基本单元 |
| **隧穿** | Quantum Tunneling，电子穿过绝缘层 |
| **KiB / MiB / GiB** | 二进制单位（IEC 标准，1 KiB = 1024 字节） |
| **KB / MB / GB** | 十进制单位（SI 标准，1 KB = 1000 字节） |
| **纠删码** | Erasure Code，分布式存储的冗余编码 |
| **元数据** | Metadata，"哪个块在哪里"的位置信息 |
| **数字黑暗时代** | Digital Dark Age，介质/设备/格式淘汰导致数据不可读 |
| **逻辑失读** | 文件完好但格式/软件绝迹，无法解读 |
| **DNA 存储** | 用合成 DNA 序列编码数字信息 |
| **碱基** | A/T/C/G，DNA 编码单位 |
| **测序** | Sequencing，读出 DNA 序列 |
