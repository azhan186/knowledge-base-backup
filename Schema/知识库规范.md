# 知识库规范 (Schema)

## 三层架构

### 第一层：Raw Sources（原始资料）
- **定位**：只属于用户的所有原始资料
- **权限**：我只读，不可修改
- **内容**：用户提供的一切原始文件、聊天记录、文档等
- **存储位置**：`/workspace/知识库/Raw_Sources/`

### 第二层：Wiki（知识库）
- **定位**：由我负责维护的知识库
- **职责**：
  - 从 Raw Sources 中读取、摘抄
  - 交叉对比验证信息
  - 整理、归纳、记录
  - 保持信息的准确性和时效性
- **存储位置**：`/workspace/知识库/Wiki/`

### 第三层：Schema（规则文件）
- **定位**：配置文档，用户和我共同维护
- **内容**：
  - Wiki 的结构规范
  - 命名约定
  - 工作流程
  - 更新机制
- **存储位置**：`/workspace/知识库/Schema/`

---

## 命名约定

- 文件夹：PascalCase 或下划线分隔
- 文件：中文命名，`.md` 格式
- 日期格式：`YYYY-MM-DD`

## 工作流程

1. 用户上传资料 → 存入 Raw Sources
2. 我分析资料 → 更新 Wiki
3. 重要规则变更 → 更新 Schema

## 强制规则

- **每次分析完成后**：必须将分析结果存入 Wiki
- 分析结果包括：摘抄、对比、验证结论、整理后的内容
- 即使是临时讨论，重要结论也应沉淀到 Wiki
- **好的问答结果应主动存回 Wiki**，不让探索结果消失在聊天历史里

## Wiki 必备文件

- **Wiki/index.md**：内容导向目录，每个页面一行的概要
- **Wiki/log.md**：添加-only 操作记录，格式：`## [YYYY-MM-DD] 操作 | 内容`

## 图片处理

- 收到的图片存入 `assets/` 目录
- 不要依赖 URL，图片必须本地保存

## Git 版本控制

- 知识库用 git 管理版本历史
- 每次重要更新后主动 commit
- commit 信息格式：`简短说明`

## 定期 Lint

- 每个一段时间让我检查 Wiki 健康度
- 检查：矛盾、过时、孤立页面、缺失交叉引用
- 发现问题主动报告
